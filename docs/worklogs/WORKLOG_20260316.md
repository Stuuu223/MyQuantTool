## 三、TradeDecisionBrain V2.0 里氏震级改造

### 3.1 问题根因

V1.0固定阈值 `entry_score_threshold=65.0` 在 `KineticCoreEngine` 公式 `×1000` 放大后，实际榜单最低分 ≥72,000，导致决策大脑对所有票无差别放行。

### 3.2 核心改造

**旧版**：固定水位线
```python
entry_score_threshold = 65.0  # 固定值
```

**新版**：动态分位数
```python
p90 = 当前榜单第90百分位
median = 中位数
入场条件 = (score >= p90) AND (score >= median × 1.5)
```

### 3.3 双通道入场

| 通道 | 条件 | 说明 |
|------|------|------|
| A (有trigger_type) | score >= p90 AND score >= median×1.5 | 有物理信号背书 |
| B (无trigger_type) | score >= p95 AND median×1.5 AND ignition_prob>=20% | 需更高标准 |

### 3.4 Bug修复

| Fix | 问题 | 修复 |
|-----|------|------|
| #1 | 固定阈值失效 | 动态分位数 |
| #2 | SELL后未清仓 | 强制调用clear_position() |
| #3 | 清仓无日志 | 写入POSITION_CLEARED |
| #4 | 阈值残留 | 改为float('inf') |

### 3.5 验证结果

```python
# 平坦榜单 [80000, 75000, 72000]
# p90≈79500, median=75000, threshold=112500
# 80000 < 112500 → WATCH ✅

# 真异动 [500000, 80000, 75000]
# p90≈416000, median=80000, threshold=120000
# 500000 >= 416000 AND >= 120000 → BUY ✅

# SELL后position=None ✅
# POSITION_CLEARED日志存在 ✅
```

### 7.4 trigger_type为空根因诊断

**问题定位**: TriggerValidator被正确调用，但传入参数为单帧数据，导致所有检测条件无法满足。

```python

trigger_signal = self.trigger_validator.check_all_triggers(
    ...
    recent_mfe_list=[mfe],           # 只有1个值，需要>=3
    price_history=[current_price],   # 只有1个值，需要>=5
    recent_volume_ratios=[ratio_stock], # 只有1个值，需要>=3
    ...
)
```

**触发条件**:
- VWAP反弹: MFE>=3个 + 价格历史>=5个 + 之前有回踩
- 阶梯突破: STAIR_DURATION_MIN=15分钟数据
- 真空点火: 量比>=3个

**结论**: 不是TriggerValidator未被调用，而是**缺少历史数据缓冲区**导致所有检测返回None。

**待办**: 需要在`run_live_trading_engine.py`中实现历史数据缓冲区(deque)，持续收集每个股票的MFE/价格/量比历史数据。

## 十二、夜间CTO审计修复 (三大修复 + Bug#1-#4)

### 12.1 CTO审计发现的核心问题

**问题一：53分钟未跑完——性能灾难**
- 根因：`mock_live_runner.py` 每分钟每只股票都在用双重for循环扫描全量Tick
- 复杂度：O(frames × stocks × ticks) = 1.87亿次迭代

**问题二：score=0买入——决策大脑逻辑漏洞**
- 根因：`top1_score >= relative_threshold` 当0 >= 0时通过
- 结果：零分保护缺失，垃圾开仓

**问题三：量纲炸了——FloatVolume单位问题**
- 根因：QMT返回的FloatVolume单位是股，代码误以为是万股
- 结果：×10000导致换手率/inflow_ratio接近0，所有票得0分

**问题四：trigger_type='none'走通道A**
- 根因：字符串`'none' != ''` 永远为True
- 结果：通道B从写好那天起从未执行过，双通道设计是假的

### 12.2 三大修复

| 修复 | 文件 | 问题 | 解决 |
|------|------|------|------|
| 量纲 | mock_live_runner.py | FloatVolume×10000 | 删除×10000，单位本就是股 |
| 零分保护 | trade_decision_brain.py | 0>=0通过入场 | 添加`top1_score<=0`跳过决策 |
