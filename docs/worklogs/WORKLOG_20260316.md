## 三、TradeDecisionBrain V2.0 里氏震级改造

### 3.1 问题根因

V1.0固定阈值 `entry_score_threshold=65.0` 在 `KineticCoreEngine` 公式 `×1000` 放大后，实际榜单最低分 ≥72,000，导致决策大脑对所有票无差别放行。

### 3.2 核心改造

**旧版**：固定水位线
```python
entry_score_threshold = 65.0  # 固定值
```

**新版**：动态分位数
```python
p90 = 当前榜单第90百分位
median = 中位数
入场条件 = (score >= p90) AND (score >= median × 1.5)
```

### 3.3 双通道入场

| 通道 | 条件 | 说明 |
|------|------|------|
| A (有trigger_type) | score >= p90 AND score >= median×1.5 | 有物理信号背书 |
| B (无trigger_type) | score >= p95 AND median×1.5 AND ignition_prob>=20% | 需更高标准 |

### 3.4 Bug修复

| Fix | 问题 | 修复 |
|-----|------|------|
| #1 | 固定阈值失效 | 动态分位数 |
| #2 | SELL后未清仓 | 强制调用clear_position() |
| #3 | 清仓无日志 | 写入POSITION_CLEARED |
| #4 | 阈值残留 | 改为float('inf') |

### 3.5 验证结果

```python
# 平坦榜单 [80000, 75000, 72000]
# p90≈79500, median=75000, threshold=112500
# 80000 < 112500 → WATCH ✅

# 真异动 [500000, 80000, 75000]
# p90≈416000, median=80000, threshold=120000
# 500000 >= 416000 AND >= 120000 → BUY ✅

# SELL后position=None ✅
# POSITION_CLEARED日志存在 ✅
```
