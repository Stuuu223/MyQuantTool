## 一、核心发现：V158回归分析颠覆V157假设

### 【致命谬误】溢出乘数公式符号错误

**代码现状** (`kinetic_core_engine.py:446`):
```python
overflow_multiplier = 1.0 + math.log10(1.0 + yesterday_vol_ratio) * 2.0
```

**V158回归分析真实物理规律**（8014样本）:
```
开盘溢价 = 6.323 + (-3.805) * log10(1 + 昨日量比) + 0.248 * MFE
```

| 维度 | 代码假设 | 回归结果 | 差异 |
|------|----------|----------|------|
| 系数符号 | **正(+2.0)** | **负(-3.805)** | ❌ 完全相反！ |
| 物理意义 | 量比高→势能继承 | 量比高→透支预警 | ❌ 完全相反！ |
| 解释力 | 假设有效 | R²=0.0398 (4%) | ❌ 几乎无效！ |

### 物理真相：量比与开盘溢价呈负相关

| 量比分位数 | 阈值 | 开盘溢价均值 |
|------------|------|--------------|
| 50th | 1.0x | +4.67% |
| 70th | 4.1x | +4.10% |
| 90th | 6.8x | +2.42% |
| 95th | 8.2x | +1.19% |

**结论**: 昨日量比越高，今日开盘溢价越低！这是"透支效应"，不是"势能继承"！

## 四、V157溢出乘数修复方案

### 当前错误代码 (L446)
```python
overflow_multiplier = 1.0 + math.log10(1.0 + yesterday_vol_ratio) * 2.0
```

### 修复方案A（基于回归）
```python
# V158回归修正：昨日量比高是透支预警，不是势能继承！
# 回归系数-3.805，取绝对值一半作为保守衰减
if yesterday_vol_ratio > 1.0:
    overflow_multiplier = max(0.5, 1.0 - math.log10(1.0 + yesterday_vol_ratio) * 0.5)
else:
    overflow_multiplier = 1.0
```

### 修复方案B（废弃跨日记忆）
```python
# R²=0.0398说明跨日预测几乎无效，废弃跨日记忆系统
overflow_multiplier = 1.0  # 统一使用基准乘数
```
