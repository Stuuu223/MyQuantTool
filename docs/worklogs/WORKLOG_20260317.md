## 二、P0任务1：卖出价格错误修复

### 2.1 问题现象

战报JSON中所有卖出价格都显示为¥5.xx，与实际持仓价格不符：
```json
{
  "stock_code": "000677.SZ",
  "buy_price": 7.40,
  "sell_price": 5.36,  // 错误！应该接近7.xx
  "pnl_pct": -27.57%
}
```

### 2.2 问题诊断

添加Debug日志追踪：
```python
logger.info(f"[DEBUG SELL] target_price={target_price} | pos.current_price={pos.current_price}")
```

发现关键线索：
```
[DEBUG SELL] target_price=4.31 | pos.current_price=0.00
```

**根因定位**：
1. 真实引擎(MockExecutionManager)拒绝了买入（涨停买不进）
2. PaperTradeEngine成功买入（零摩擦）
3. 决策大脑状态显示`held_stock_code=000677.SZ`
4. 但`held_price`从真实引擎获取，真实引擎没有持仓
5. 导致`held_price`使用了错误的默认值

### 2.3 修复方案

**文件**: `tools/mock_live_runner.py`

```python
# ── Phase B: 决策大脑
# 【P0修复】从正确的引擎获取held_price
held_price = 0.0
brain_held_code = self.decision_brain.held_stock_code

if brain_held_code:
    # 优先从真实引擎获取
    if brain_held_code in self.execution_manager.positions:
        pos = self.execution_manager.positions[brain_held_code]
        held_price = pos.current_price if pos.current_price > 0 else pos.entry_price
    # 其次从PaperTrade引擎获取
    elif brain_held_code in self.paper_engine.positions:
        held_price = self.paper_engine.positions[brain_held_code].get('current_price', 0.0)
        if held_price <= 0:
            held_price = self.paper_engine.positions[brain_held_code].get('cost_price', 0.0)
    # 最后从榜单获取（最可靠）
    target = next((t for t in enriched_targets if t['code'] == brain_held_code), None)
    if target and target.get('price', 0) > 0:
        held_price = target['price']
```

### 2.4 验证结果

修复后卖出价格正确：
```json
{
  "stock_code": "000677.SZ", 
  "buy_price": 7.40,
  "sell_price": 7.85,  // 正确！
  "pnl_pct": +6.08%
}
```

## 三、P0任务2：median_score全为0修复

### 3.1 问题现象

战报中所有决策的`median_score`字段都是0：
```json
{
  "action": "SELL",
  "median_score": 0,  // 错误！
  "p90_threshold": 0   // 错误！
}
```

### 3.2 问题诊断

查看`trade_decision_brain.py`的`on_new_frame`方法：

```python
if decision.action == 'BUY':
    decision.p90_threshold = p90_threshold
    decision.median_score = median_score
    return decision
```

**根因定位**：
- BUY分支正确填充了`median_score`和`p90_threshold`
- 但SELL分支和HOLD分支没有填充这些字段
- 导致这些字段使用默认值0

### 3.3 修复方案

**文件**: `logic/execution/trade_decision_brain.py`

在SELL分支添加：
```python
decision.action = 'SELL'
decision.stock_code = self.held_stock_code
decision.price = held_stock_current_price
decision.reason = exit_decision[1]
decision.score = self.entry_score
# 【P0修复】SELL分支也填充p90和median（使用缓存值）
decision.p90_threshold = self._last_frame_p90
decision.median_score = self._last_frame_median
```

在HOLD分支添加：
```python
decision.action = 'HOLD'
decision.stock_code = self.held_stock_code
decision.price = held_stock_current_price
decision.score = self.entry_score
# 【P0修复】HOLD分支也填充p90和median
decision.p90_threshold = self._last_frame_p90
decision.median_score = self._last_frame_median
```

## 五、P2任务5：30分钟冷静期防止车轮战

### 5.1 问题现象

系统出现"车轮战"行为：
- 09:45 卖出股票A
- 09:46 立即买入股票B
- 09:55 卖出股票B
- 09:56 立即买入股票C

这种高频交易不符合右侧追涨策略的设计初衷。

### 5.2 修复方案

**文件**: `logic/execution/trade_decision_brain.py`

新增属性：
```python
# 【P2修复】卖出后冷静期：防止车轮战
self._last_sell_time: Optional[datetime] = None
self._sell_cooldown_minutes: int = config.get('sell_cooldown_minutes', 30)
```

在`clear_position`中记录卖出时间：
```python
def clear_position(self):
    self.held_stock_code = None
    self.entry_score = None
    self.entry_time = None
    self._last_sell_time = datetime.now()  # 【P2修复】记录卖出时间
```

在`_should_enter`中检查冷静期：
```python
def _should_enter(self, current_time: datetime, ...):
    # 【P2修复】冷静期检查
    if self._last_sell_time is not None:
        minutes_since_sell = (current_time - self._last_sell_time).total_seconds() / 60
        if minutes_since_sell < self._sell_cooldown_minutes:
            remaining = self._sell_cooldown_minutes - minutes_since_sell
            logger.debug(f"[冷静期] 卖出后{minutes_since_sell:.0f}分钟，还需等待{remaining:.0f}分钟")
            return None
```
