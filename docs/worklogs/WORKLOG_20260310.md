## Phase 4: V66绝地反击 - 万股量纲根治

### 4.1 问题诊断（核心根因）
**症状**: V64后出现空榜

**根因链条**:
1. QMT返回的`float_volume_shares`单位是**万股**，不是股！
2. 志特新材流通盘1.5亿股 → QMT返回15000（万股）
3. `float_market_cap = 15000 * 10 = 15万`（实际应是15亿！）
4. `inflow_ratio = 5000万/15万 = 33333%`（爆炸！）
5. V64切掉了`*10000`的救命稻草，导致空榜

### 4.2 修复内容
1. **量纲升维**: 市值<2亿时强制×10000升维
2. 恢复`calibrated_float_market_cap`变量兼容后续代码
3. 清理重复计算`inflow_ratio_pct`的代码
4. **MFE制衡下限保护**: `max(0.5, mfe)`，给重卡留体面

### 4.3 物理法则
- A股最小流通市值不可能<2亿
- 如果市值<2亿，100%是万股单位
- MFE<1.0时最惨打5折，不至于削成0.3

## 核心教训

### V64失误分析
> V64的`*10000`不是"市值放大偏见"，而是"万股→股"的救命稻草！
> 物理引擎必须无状态，但数据源头量纲混乱时，必须在入口处清洗！

### 量纲铁律
| 数据 | 单位 | 说明 |
|------|------|------|
| amount | 元 | QMT成交额，绝对准确 |
| volume | 手(100股) | 需要×100转股 |
| float_volume | 万股 | **需要×10000转股！** |
| price | 元 | 绝对准确 |

### 计算公式
```python
# 流通市值（元）
float_market_cap = float_volume_shares * price

# 量纲升维（A股最小流通市值不可能<2亿）
if float_market_cap > 0 and float_market_cap < 200000000:
    float_market_cap = float_market_cap * 10000.0
```
