# 临时文件目录

**用途**: 存放临时文件，用完即删

## 目录结构

```
temp/
├── tests/              # 临时测试脚本
├── debug/              # 调试输出
├── logs/               # 临时日志
├── data/               # 临时数据
└── README.md           # 本文件
```

## 使用规范

### 命名规范

**临时文件**: `[用途]_temp_[日期].[ext]`

示例：
- `test_redis_temp_20260202.py`
- `debug_auction_temp_20260202.txt`
- `temp_data_20260202.json`

### 使用原则

1. **及时清理**: 任务完成后立即删除临时文件
2. **不提交 Git**: 此目录中的文件不应提交到版本控制
3. **按类型分类**: 将临时文件放到对应的子目录中
4. **定期清理**: 每周清理一次此目录

### 清理命令

```bash
# 清理所有临时文件
del /q temp\*

# 清理特定子目录
del /q temp\tests\*
del /q temp\debug\*
```

## 注意事项

- 此目录不应包含重要数据
- 定期检查并清理过期文件
- 不要在此目录中存放需要长期保留的文件
- 如需长期保留，请移动到适当的目录（如 data/、docs/ 等）
