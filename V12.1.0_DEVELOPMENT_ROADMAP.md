**问题1：高盈亏比、低胜率**
- 22.73% 的胜率意味着每做 5 笔交易就要忍受 4 次止损

- 66 次/月 的交易频率可能过度交易
- 缺乏持仓时间限制，容易被洗盘

**问题3：止损占主导**
- 77.27% 的交易因止损卖出
- 需要优化止损策略或提高买入质量

## 🎯 CTO 核心裁决与方向指引

### 一、数据与基础设施裁决

| 领域 | CTO裁决 | 具体要求 |
|------|---------|----------|
| **QMT Level-1** | 必须使用Level-1推断 | 基于Tick数据推断资金流，禁止降级到分钟K |
| **资金流数据** | 抽象层封装 | 策略层必须通过ICapitalFlowProvider接口访问 |
| **数据源策略** | 自动降级 | Level-2 → Level-1 → DongCai，不强制依赖Level-2 |
| **数据新鲜度** | 3秒刷新率 | Tick数据延迟3秒，比分钟K快20倍 |

**CTO核心论断**：
> "QMT提供的是Level-1数据（3秒刷新），能否准确计算'1分钟内主动买入>1000万'？是否需要升级到Level-2数据？"

**CTO裁决**：够用，但需算法补偿。Level-2虽然逐笔，但成本高且接口复杂。Level-1的3秒快照配合差分算法足够捕捉"脉冲式"资金。

### 二、策略逻辑裁决

| 策略模块 | CTO裁决 | 具体要求 |
|----------|---------|----------|
| **假攻击识别** | 必须区分真攻击和假攻击 | 检测持续流入、量价关系、买卖盘力量 |
| **阈值设定** | 必须动态调整 | 根据市值比例设定阈值，禁止固定值 |
| **ratio决策树** | 不可替代 | 买入决策必须基于ratio（主力净流入/流通市值） |
| **诱多检测** | 不可省略 | 单日暴量+隔日反手、游资突袭、对倒 |

**CTO核心论断**：
> "1分钟内买入1000万，是真攻击还是诱多？如何避免追在最高点被套？"

**CTO建议方案**：
```python
class TrueAttackDetector:
    def is_true_attack(self, code, flow_data):
        # 特征1：持续流入（不是一闪而过）
        if not self._check_sustained_inflow(flow_data):
            return False
        
        # 特征2：价格上涨伴随放量（不是对倒）
        if not self._check_volume_price_relationship(flow_data):
            return False
        
        # 特征3：买盘力量>卖盘力量（主力真买）
        if flow_data['main_sell'] > flow_data['main_buy']:
            return False
        
        # 特征4：不是尾盘偷袭（避免尾盘拉升出货）
        if self._is_last_15_minutes():
            return False
        
        return True
```

### 三、系统架构裁决

| 架构维度 | CTO裁决 | 具体要求 |
|----------|---------|----------|
| **事件驱动** | 强制要求 | 检测到事件才扫描，不固定间隔扫描 |
| **三把斧体系** | 不可删减 | 防守斧+时机斧+资金斧，缺一不可 |
| **板块共振** | 入场前提 | Leaders≥3 + Breadth≥35%，否则降级到观察池 |
| **状态管理** | 指纹对比 | 只有状态变化才保存快照 |

**CTO核心论断**：
> "如何平衡实时性和准确性？1秒级监控会消耗大量资源。是否需要分层监控（高频扫描+低频确认）？"

**CTO裁决**：分层架构。
- Layer 1（快速扫描）：全市场5000只，只看涨速>1%
- Layer 2（深度确认）：对Layer 1筛出来的20-50只，进行毫秒级Tick分析

### 四、核心战略裁决

| 战略方向 | CTO裁决 | 具体要求 |
|----------|---------|----------|
| **预测vs爬榜** | 预测优先 | 顽主杯是训练集，不是信号源；预测谁会上榜 |
| **实战导向** | 数据为王 | 没有数据验证，所有逻辑都是纸上谈兵 |

**CTO核心论断**：
> "顽主杯榜单本身是T-1数据，已经滞后。为什么还要'实时版'？真正的价值不是跟随，而是**预测谁上榜**。"

## 🚀 V12.1.0 优化方案（三大过滤器）

### 过滤器1：风口过滤器（板块共振增强）

**CTO观点**：
> "板块共振是入场前提，Leaders≥3 + Breadth≥35%，否则降级到观察池。"

**拾荒网知识**：
> "第一板斧：判断风口 - 市场主流热点、题材方向，板块共振判断（Leaders≥3, Breadth≥35%）。"

**实现方案**：
```python
class WindFilter:
    """
    风口过滤器：基于板块共振增强
    """
    
    def check_sector_resonance(self, stock_code):
        """
        检查板块共振状态
        """
        sector = get_sector(stock_code)
        
        # 条件1：板块内涨停股≥3只
        limit_up_count = count_limit_up_in_sector(sector)
        
        # 条件2：板块内上涨股票占比≥35%
        up_count = count_up_stocks_in_sector(sector)
        total_count = count_total_stocks_in_sector(sector)
        breadth = up_count / total_count if total_count > 0 else 0
        
        # 条件3：板块指数连续3日资金净流入（新增）
        sector_index_flow = get_sector_index_flow(sector, days=3)
        sustained_inflow = all(flow > 0 for flow in sector_index_flow)
        
        # 共振判定
        is_resonance = (
            limit_up_count >= 3 and
            breadth >= 0.35 and
            sustained_inflow
        )
        
        return {
            "is_resonance": is_resonance,
            "limit_up_count": limit_up_count,
            "breadth": breadth,
            "sustained_inflow": sustained_inflow,
            "resonance_score": self._calculate_resonance_score(
                limit_up_count, breadth, sustained_inflow
            )
        }
    
    def _calculate_resonance_score(self, limit_up_count, breadth, sustained_inflow):
        """
        计算共振分数（0-1）
        """
        # 权重设置
        w_limit_up = 0.4
        w_breadth = 0.3
        w_sustained = 0.3
        
        # 标准化
        score_limit_up = min(limit_up_count / 10, 1.0)
        score_breadth = min(breadth / 0.5, 1.0)
        score_sustained = 1.0 if sustained_inflow else 0.0
        
        # 加权求和
        resonance_score = (
            w_limit_up * score_limit_up +
            w_breadth * score_breadth +
            w_sustained * score_sustained
        )
        
        return resonance_score
```

**预期效果**：过滤掉 30%-50% 的无效交易，大幅提升胜率。

### 过滤器2：动态止损/止盈

**CTO观点**：
> "止盈根据信号去止盈，如主力资金大量流出等，上涨动力已经缺乏。不是死板的30-50%。"

**拾荒网知识**：
> "情绪下降期休息，不要强行操作。"

**实现方案**：
```python
class DynamicStopManager:
    """
    动态止损/止盈管理器
    """
    
    def calculate_stop_loss(self, stock_code, buy_price, current_price, flow_data):
        """
        动态止损计算
        """
        # 基础止损：-5%
        base_stop_loss = buy_price * 0.95
        
        # 条件1：主力资金大量流出 → 提前止损
        if flow_data['main_net_inflow'] < -50000000:  # 流出>5000万
            return buy_price * 0.97  # -3%止损
        
        # 条件2：情绪周期处于退潮/冰点 → 提前止损
        sentiment_stage = get_sentiment_cycle_stage()
        if sentiment_stage in ['退潮', '冰点']:
            return buy_price * 0.96  # -4%止损
        
        # 条件3：板块共振失效 → 提前止损
        sector_resonance = check_sector_resonance(stock_code)
        if not sector_resonance['is_resonance']:
            return buy_price * 0.97  # -3%止损
        
        return base_stop_loss
    
    def calculate_take_profit(self, stock_code, buy_price, current_price, flow_data):
        """
        动态止盈计算
        """
        profit_rate = (current_price - buy_price) / buy_price
        
        # 基础止盈：+30%
        base_take_profit = buy_price * 1.30
        
        # 条件1：主力资金大量流出 → 提前止盈
        if flow_data['main_net_inflow'] < -50000000:
            return current_price  # 立即止盈
        
        # 条件2：上涨动力丧失（股价与量比背离） → 提前止盈
        if self._check_price_volume_divergence(flow_data):
            return current_price  # 立即止盈
        
        # 条件3：情绪周期处于高潮 → 延长止盈
        sentiment_stage = get_sentiment_cycle_stage()
        if sentiment_stage in ['高潮', '主升'] and profit_rate > 0.20:
            return buy_price * 1.50  # +50%止盈
        
        # 条件4：板块共振增强 → 延长止盈
        sector_resonance = check_sector_resonance(stock_code)
        if sector_resonance['is_resonance'] and profit_rate > 0.25:
            return buy_price * 1.45  # +45%止盈
        
        return base_take_profit
```

**预期效果**：减少无效止损，提高单笔盈利，平滑收益曲线。

### 过滤器3：竞价强弱校验

**CTO观点**：
> "竞价预期规则：明牌焦点股，竞价不能超预期 → 打折扣；明牌股，竞价超预期 → 不要轻易卖。"

**拾荒网知识**：
> "量比排序使用：9:25竞价出来后，9:30开盘后15分钟内完成。量比排序 + 涨幅排序配合使用。"

**实现方案**：
```python
class AuctionStrengthValidator:
    """
    竞价强弱校验器
    """
    
    def validate_auction(self, stock_code, auction_data, is_focus_stock=False):
        """
        验证竞价强弱
        """
        # 条件1：高开幅度
        high_open_rate = (auction_data['open_price'] - auction_data['prev_close']) / auction_data['prev_close']
        
        # 条件2：量比
        volume_ratio = auction_data['volume_ratio']
        
        # 条件3：竞价爆量（竞价最后一刻是红柱）
        is_auction_volume_surge = self._check_auction_volume_surge(auction_data)
        
        # 竞价强弱判定
        if is_focus_stock:
            # 明牌焦点股：竞价不能超预期 → 打折扣
            if high_open_rate > 0.05:  # 高开>5%
                return {
                    "is_valid": False,
                    "reason": "明牌焦点股，竞价超预期，打折扣",
                    "action": "WATCH"
                }
        else:
            # 非明牌股：竞价超预期 → 不要轻易卖
            if high_open_rate > 0.05 and volume_ratio > 2.0:
                return {
                    "is_valid": True,
                    "reason": "竞价爆量跳空高开",
                    "action": "STRONG_BUY",
                    "confidence": 0.85
                }
        
        # 基础判定：量比+涨幅配合
        if volume_ratio > 1.5 and high_open_rate > 0.02:
            return {
                "is_valid": True,
                "reason": "量比排序+涨幅排序配合",
                "action": "BUY",
                "confidence": 0.70
            }
        
        return {
            "is_valid": False,
            "reason": "竞价强度不足",
            "action": "WATCH"
        }
```

**预期效果**：提高开盘阶段选股精度，减少竞价陷阱。
