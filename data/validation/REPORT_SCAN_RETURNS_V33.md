## 一、背景：从高分低能到照妖镜

### 1.1 问题的发现

在CTO V31-V51的开发历程中，团队发现一个诡异现象：

| 案例 | 分数 | 最大收益 | 问题诊断 |
|------|------|----------|----------|
| 600714.SH | 9072分 | +14.40% | 高分低能！ |

**四大陷阱根因**：
1. **ratio_stock分母灾难**：僵尸股flow_5min_median基数极低，稍点火就百倍放量
2. **inflow_ratio_pct体量幻觉**：微盘股800万流入8亿盘=1%拿满分
3. **day_strength绝对涨幅缺失**：水下股在最高点也得满分
4. **乘数效应无脑放大**：底数小×乘数大=虚假繁荣

### 1.2 CTO Boss核心洞察

> **"90分天花板不仅压制龙头，更是杂毛的保护伞！"**  
> 砸碎上限让真龙现形，照妖镜让杂毛现原形！

### 1.3 V33照妖镜核心设计

| 算子 | 逻辑 | 效果 |
|------|------|------|
| **杂毛现形器** | change_pct>8.5%且未封板 → ×0.3惩罚 | 高位烂板现原形 |
| **真龙升天器** | 封板且封单>0 → ×(1.5+queue_bonus)奖励 | 封板强度定价 |
| **砸碎阻尼上限** | min(sustain_ratio, 10) → max(sustain_ratio, -1) | 向上无顶 |
| **MIN_BASE_FLOW** | flow_5min_median取max(x, 200万) | 僵尸股分母托底 |

## 五、问题诊断与改进方向

### 5.1 核心问题：平均最终收益为负

**现象**：-2.13%的平均最终收益意味着"买榜单股票会亏钱"。

**根因分析**：

| 可能原因 | 证据 | 概率 |
|----------|------|------|
| 照妖镜过于严格 | 88只全被识别为非封板 | 高 |
| 时间冻结过早 | 09:45错过涨停时间窗 | 高 |
| 样本期间市场弱势 | 正收益比例仅29.5% | 中 |
| 封板检测失效 | askPrice1数据可能缺失 | 中 |

### 5.2 改进建议

| 改进项 | 具体措施 | 预期效果 |
|--------|----------|----------|
| **时间窗口调整** | 冻结时间从09:45改为11:00 | 捕捉更多封板信号 |
| **封板检测增强** | 结合日K涨停价判断封板 | 不依赖Tick askPrice |
| **照妖镜阈值校准** | 8.5%阈值动态化 | 避免误杀 |
| **止盈机制** | 最大收益>15%触发跟踪止损 | 锁定浮盈 |

### 5.3 V33 vs V51 定位差异

| 维度 | V33照妖镜 | V51里氏震级 |
|------|-----------|-------------|
| 目标 | 过滤杂毛 | 识别真龙 |
| 分数范围 | 窄(94-1208) | 宽(1215-9072) |
| 选股逻辑 | 防守优先 | 进攻优先 |
| 适用场景 | 熊市/震荡市 | 牛市/主升浪 |
| 正收益比例 | 29.5% | 51.2% |

**结论**：V33是防守型引擎，V51是进攻型引擎。当前测试数据可能更适合V51。

## 六、技术细节

### 6.1 V33照妖镜公式

```python
# A. MIN_BASE_FLOW绝对物理基线
MIN_BASE_FLOW = 2_000_000.0  # 200万
safe_median = max(flow_5min_median_stock, MIN_BASE_FLOW)
ratio_stock = safe_flow_5min / safe_median
ratio_stock = max(0.01, ratio_stock)  # 向上无压制！

# B. 砸碎sustain_ratio上限
sustain_ratio = max(-1.0, sustain_ratio)  # 只防负溢出

# C. 杂毛现形器
change_pct = (price - prev_close) / prev_close
if change_pct > 0.085 and not is_limit_up:
    multiplier *= 0.3  # 打7折！

# D. 真龙升天器
if is_limit_up and limit_up_queue_amount > 0:
    queue_bonus = min(3.0, limit_up_queue_amount / 100_000_000.0)
    multiplier *= (1.5 + queue_bonus)
```

### 6.2 封板检测逻辑

```python
# Tick数据解析
ask_price1 = tick.get('askPrice1', 0.0) or 0.0
bid_price1 = tick.get('bidPrice1', 0.0) or 0.0
bid_vol1 = (tick.get('bidVol1', 0) or 0) * 100  # 手转股

# 封板判断：卖一为空，买一有量
is_limit_up = (ask_price1 == 0.0 and bid_price1 > 0.0)

# 封单金额
limit_up_queue_amount = (bid_price1 * bid_vol1) if is_limit_up else 0.0
```
