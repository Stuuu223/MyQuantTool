## 一、起源：从V50到V51的进化

### 1.1 问题发现

在2026-03-07的CTO审计中，团队发现三个致命问题：

| 问题 | 现象 | 影响 |
|------|------|------|
| 300只硬编码 | `[:300]`截断 | 粗筛泛滥，尖刀淹没 |
| 90分天花板 | `min(30, ...)`截断 | 龙头无法脱颖而出 |
| 90th分位丢失 | 动态量比防线缺失 | 榜单质量下降 |

### 1.2 Boss核心洞察

> **"观察池数量就是天然水位仪"**  
> 废除所有min()截断，用乘法动能让龙中龙脱颖而出！

### 1.3 V50修复清单

| 修复项 | 修复前 | 修复后 |
|--------|--------|--------|
| 时间冻结 | 15:00(尾盘腰斩) | **09:45(无衰减)** |
| kinetic_score | min(30, inflow/1.0*30) | **min(inflow*5.0, 40)** |
| 90th分位防线 | 缺失 | **已恢复** |
| [:300]硬编码 | 存在 | **已删除** |

### 1.4 V51里氏震级模型

**核心公式变革**：

```python
# V50加法凑分（有上限）
kinetic_score = min(inflow_ratio * 30, 30)  # 上限30分
potential_score = min(sustain_ratio * 10, 30)  # 上限30分
final_score = kinetic_score + potential_score + base_score  # 上限~180分

# V51里氏震级模型（无上限）
base_power = inflow_ratio_pct * ratio_stock * price_strength
final_score = base_power * time_multiplier  # 无上限！
```

**物理意义**：
- `inflow_ratio_pct`: 资金流入占流通市值百分比
- `ratio_stock`: 相对放量倍数（当前量比）
- `price_strength`: 价格推力（日内价格位置）
- 三者相乘，爆发力无上限

## 九、技术细节

### 9.1 V51里氏震级公式

```python
# 第一步：乘法动能
if inflow_ratio_pct > 0:
    base_power = inflow_ratio_pct * ratio_stock * price_strength
else:
    base_power = abs(inflow_ratio_pct) * 0.5  # 基础分，不爆发

# 第二步：涨停奖励
base_power += 40.0 if price > prev_close and price_strength >= 0.99 else 0.0

# 第三步：VWAP惩罚
base_power = max(0.0, base_power - vwap_penalty)

# 第四步：时间乘数
multiplier = 1.0  # 冻结时间09:45，无衰减

# 第五步：最终得分（无上限里氏震级！）
final_score = round(base_power * multiplier, 2)
```

### 9.2 关键参数

| 参数 | 值 | 说明 |
|------|-----|------|
| frozen_time | 09:45 | 时间冻结，无尾盘衰减 |
| 打分阈值 | >=50分 | 入选榜单最低分 |
| 纯度阈值 | >-50% | 过滤假账 |
| TopN | 20只 | 每日榜单上限 |
