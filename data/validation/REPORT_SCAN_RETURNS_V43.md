## 核心修复清单

### Phase 1: 高水位动态止盈（Trailing Stop）
- **问题**：原有止盈逻辑过于简单，无法动态追踪最高价
- **方案**：根据分数阶级和利润率设置不同回撤容忍度
  - 真龙阶级（≥3000分）：赚30%以上允许15%回撤，赚15%以上允许10%回撤
  - 平民阶级（<3000分）：赚20%回撤10%必走，赚10%回撤6%落袋
- **关键修复**：trailing_stop不能低于entry_price*1.02，确保止盈逻辑能正确触发

### Phase 2: 盘中跳空逃生通道
- **问题**：隔夜跳空低开可能直接击穿止盈线
- **方案**：检测开盘价是否直接击穿trailing_stop
  - 若open <= trailing_stop，用开盘价退出（跳空被核）
  - 否则用trailing_stop*0.985（含滑点）

### Phase 3: 一字板换手率惩罚
- **问题**：一字涨停板换手率极低，可能是庄股操纵
- **方案**：在kinetic_core_engine.py添加turnover_multiplier
  - 换手率<2%：分数打5折（一字板惩罚）
  - 换手率5%-20%：分数×1.5（健康换手真龙）

### Phase 4: 动态涨停阈值
- **问题**：主板10%/创业板20%/北交所30%混用同一阈值
- **方案**：根据股票代码前缀动态计算阈值
  - 30/68开头（创业板/科创板）：19.5%
  - 8/4开头（北交所）：29.5%
  - 其他（主板）：9.5%

## 关键发现：max_return计算陷阱

**问题**：300164.SZ显示47.82%最大收益，但实际最终收益-2.11%

**根因分析**：
- 入场日20260225：close=14.68, high=15.29
- 入场日trailing_stop = 15.29 * 0.90 = 13.76
- **但13.76 < 14.68（入场价）！**
- 这意味着止盈条件永远不会触发

**修复后**：
- trailing_stop = max(13.76, 14.68*1.02) = 14.97
- 第二天(20260226)：low=14.37 < 14.97，触发止盈

**但仍然-2.62%的原因**：
- 实际触发的是VWAP止损，不是高水位止盈
- max_return的47.82%包含了退出后继续上涨的数据（后续涨到21.70）
- 这是**事后数据污染**，不是真实持仓收益

## 待解决问题

### P0: max_return事后数据污染
- 当前逻辑：max_price计算包括所有数据，不区分是否已退出
- 影响：报告中max_return是虚高的，误导性极强
- 方案：记录exit_loc，只计算entry到exit之间的max_price

### P1: 分数倒挂依然严重
- 高分股平均收益反而更低
- 需要重新审视打分逻辑

## 结论

V43的四大修复虽然逻辑正确，但由于核心问题（max_return事后数据污染、分数倒挂）未解决，回测结果反而变差。

**下一步行动**：
1. 修复max_return计算逻辑
