## 执行摘要

本研究通过扫描2022-2025年（约3年）A股全市场日K数据，识别出38,592个暴力拉升样本，反推"右侧起爆"的量化特征。

**核心结论**：
1. **两种模式并存**：慢牛（20-30天涨20-30%）vs 暴力拉升（3天涨50%+）
2. **起爆换手率差异显著**：慢牛模式起爆换手2-3%，暴力模式起爆换手12-13%
3. **config建议调整**：sweet_spot应分层设计，而非单一区间

---

# 一、研究背景与目标

## 1.1 问题起源

- **时间**：2026年2月~3月
- **触发事件**：换手率验证发现"低换手=高收益"，但实际交易中存在"高换手暴力拉升"案例
- **核心困惑**：为什么换手率研究与暴力拉升案例存在矛盾？

## 1.2 研究目标

从历史数据反推"右侧起爆"的量化特征，回答以下核心问题：

1. 真正的主升段起爆时，换手率在什么区间？
2. 不同涨幅强度（20%~300%），天数分布有何规律？
3. 如何解释"低换手=高收益"与"暴力拉升=高换手"的矛盾？

## 1.3 研究假设

- H1：不同涨幅档位，起爆换手率存在显著差异
- H2：存在两种截然不同的拉升模式
- H3：原config的单一甜点位无法覆盖所有模式

# 二、数据来源与研究方法

## 2.1 数据来源

| 项目 | 说明 |
|------|------|
| 数据源 | 迅投QMT（XtQuant）- 本地客户端，端口58610 |
| 数据类型 | 日K线（开高低收量）+ 流通股本（FloatVolume） |
| 时间窗口 | 2022-01-25 ~ 2026-03-05（约3年，~750个交易日） |
| 股票池 | 沪深A股（不含北交所），5,191只 |
| 样本量 | 38,592个暴力拉升事件 |

**已知局限**：
- 股票池为"当前时点A股"，存在幸存者偏差
- FloatVolume使用当前值，对2022-2024数据有轻微影响（<5%偏差）

## 2.2 核心指标定义

### 2.2.1 换手率计算

```
换手率 = 成交量（手）× 100 / 流通股本（股）× 100%
```

### 2.2.2 暴力拉升识别逻辑

**筛选条件**：
- 任意3~30日涨幅达到20%~300%
- 期间最大回撤 <25%
- 不预设固定天数，让数据自然分布

**扫描算法**：
```python
for 每个可能起点:
    向前看3~30天:
        if 涨幅达标 AND 回撤可控:
            记录为有效样本
```

## 2.3 研究方法

采用"先筛选样本，再反推规律"的归纳法：
1. 全市场扫描，找出所有符合条件的暴力拉升样本
2. 按涨幅/天数分桶，统计每档的换手特征
3. 交叉分析（涨幅 x 天数），找出模式差异

# 四、核心发现

## 4.1 两种截然不同的模式

### 模式A：慢牛稳健型（占99%+样本）

| 特征 | 数值 |
|------|------|
| 涨幅区间 | 20-40% |
| 天数区间 | 15-30天 |
| 起爆换手率 | 2-5% |
| 主升段换手 | 2-4% |
| 涨停板数 | 0-1个 |

**典型特征**：低换手起爆，温和放量，稳健上涨

### 模式B：暴力拉升型（占<1%样本）

| 特征 | 数值 |
|------|------|
| 涨幅区间 | 50-100%+ |
| 天数区间 | 3-5天 |
| 起爆换手率 | 12-15% |
| 主升段换手 | 11-18% |
| 涨停板数 | 1-2个 |

**典型特征**：高换手起爆，持续放量，快速拉升

## 4.2 模式矛盾解释

**问题**：为什么换手率研究表明"低换手=高收益"，但暴力拉升却是"高换手"？

**解答**：
1. **研究对象不同**：
   - 换手率研究：观察所有股票的"未来收益"，发现低换手股票表现更好
   - 暴力拉升研究：只观察"已经暴涨"的股票，反推其起爆特征

2. **样本分布不同**：
   - 换手率研究样本：~14M个，涵盖所有股票
   - 暴力拉升样本：38,592个，仅占0.3%

3. **结论兼容**：
   - 低换手股票整体表现更好（胜率更高）
   - 但极端暴力拉升（<1%）需要高换手触发

# 五、策略建议

## 5.1 Config分层设计建议

基于本研究，建议将单一的`sweet_spot`参数改为分层设计：

```json
{
  "mode_settings": {
    "slow_bull": {
      "description": "慢牛稳健型（15-30天涨20-40%）",
      "turnover_range": [1.0, 5.0],
      "target_days": [15, 30],
      "target_return": [20, 40],
      "sample_pct": 99.38
    },
    "violent_surge": {
      "description": "暴力拉升型（3-5天涨50%+）",
      "turnover_range": [10.0, 15.0],
      "target_days": [3, 5],
      "target_return": [50, 100],
      "sample_pct": 0.13,
      "risk_warning": "样本极少，高风险高收益"
    }
  },
  "death_trap": {
    "turnover_rate_max": 70.0,
    "warning": "换手>70%为死亡线"
  }
}
```

## 5.2 分模式筛选逻辑

### 慢牛模式筛选（推荐主策略）

```python
def is_slow_bull_ignition(stock, today):
    # 条件1：换手在甜点区（1-5%）
    if not (1.0 <= today_turnover <= 5.0):
        return False
    
    # 条件2：价格在MA20上方
    if close < ma20:
        return False
    
    # 条件3：起爆前20日换手<5%
    if avg_turnover_20d > 5.0:
        return False
    
    return True
```

### 暴力拉升模式筛选（高风险）

```python
def is_violent_surge_ignition(stock, today):
    # 条件1：换手在高位（10-15%）
    if not (10.0 <= today_turnover <= 15.0):
        return False
    
    # 条件2：当日涨幅>5%
    if today_return < 5.0:
        return False
    
    # 条件3：有涨停板迹象
    if not is_near_limit_up(stock, today):
        return False
    
    # 风险警告
    return True, "高风险模式，样本稀缺"
```

## 5.3 风险隔离规则

```python
def is_death_trap(stock, today):
    # 死亡陷阱1：单日换手>70%
    if today_turnover > 70:
        return True, "超高换手死亡线"
    
    # 死亡陷阱2：暴力拉升失败（高换手+次日跌）
    if today_turnover > 15 and next_day_return < -5:
        return True, "暴力拉升失败"
    
    return False, None
```

# 六、研究局限与后续计划

## 6.1 当前研究局限

### 6.1.1 已知偏差

1. **幸存者偏差**：股票池为当前A股，已退市股票未纳入
2. **浮动分母问题**：FloatVolume使用当前值，未考虑历史增发/回购
3. **停牌未剔除**：复牌后"虚假涨幅"可能污染样本

### 6.1.2 未覆盖维度

1. **流动性约束**：未评估大资金介入后对换手的扰动
2. **涨跌停限制**：未考虑连续涨停无法买入的实战问题
3. **分钟K精度**：无法识别盘中起爆时刻

3. **分钟K优化**：识别盘中起爆时刻
