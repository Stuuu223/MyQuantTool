## Phase 4: 关键发现

### 4.1 max_return事后数据污染（P0）
- **现象**: 300164.SZ max_return=47.82%，但实际T+1就退出
- **根因**: max_price计算包括所有数据，不区分是否已退出
- **影响**: 报告中max_return严重虚高，误导性极强
- **方案**: 记录exit_loc，只计算entry到exit之间的max_price

### 4.2 分数倒挂问题（P1）
- **现象**: 高分股平均收益反而更低
- **验证**: 需要对比Score Top10 vs Return Top10

### 4.3 动态止盈效果不佳
- **现象**: 只有5只股票触发高水位止盈
- **原因**: 大部分在达到止盈条件前就触发VWAP止损

## Phase 6: V45 Tick级全息沙盘重构

### 6.1 CTO震怒审计
CTO发现之前报告中声称修改的代码实际上并未生效，并指出三大隐患：
1. 回测脚本与实盘引擎"脑裂"（SSOT铁律坍塌）
2. 时间机器颗粒度坍塌（Tick数据降维成日线）
3. L1放量滞涨探针的"绝对值陷阱"

### 6.2 核心修复执行

**战区一：sustain_ratio分母倒挂修复**
- **文件**: `logic/strategies/kinetic_core_engine.py`
- **旧逻辑**: `sustain_ratio = flow_next_10min / flow_5min`（用前5分钟当分母）
- **新逻辑**: `sustain_ratio = flow_15min / safe_median_15min`（用历史中位数当基准）
- **物理意义**: 不再惩罚早盘爆量真龙

**战区二：回测引擎Tick级升维**
- **文件**: `tests/test_scan_returns.py`
- **旧逻辑**: `period='1d'` 日线数据，只能用收盘价判断
- **新逻辑**: `period='tick'` Tick级数据，盘中实时VWAP止损
- **关键发现**: QMT Tick数据的amount/volume已经是累计值，不需要cumsum

**战区三：VWAP计算优化**
- Tick级动态VWAP = 累计成交额 / 累计成交量
- 价格跌破VWAP 1%立即斩仓
