## 📋 核心工作流程

### 1. Git 提交和推送（强制执行）

**触发条件**:
- 每次更新迭代
- 每次代码修改
- 每次功能添加
- 每次 Bug 修复

**执行步骤**:
```bash
# 1. 检查 Git 状态
git status

# 2. 添加所有更改
git add -A

# 3. 创建提交（使用详细的 commit message）
git commit -m "描述性标题

## 核心改进
- 改进点 1
- 改进点 2

## 测试结果
- 测试项 1: ✅ 通过
- 测试项 2: ✅ 通过

Author: iFlow CLI
Date: YYYY-MM-DD"

# 4. 推送到 GitHub
git push origin master
```

**Commit Message 规范**:
- 标题：简明扼要，描述主要变更
- 正文：详细描述改进内容、测试结果、系统状态
- 格式：使用 Markdown 格式，包含适当的标题和列表

## ⚠️ 注意事项

### 1. 大文件警告
- GitHub 推荐最大文件大小：50 MB
- 如果文件超过此大小，会收到警告但仍可推送
- 建议使用 Git LFS 管理大文件

### 2. 分支管理
- 默认分支：`master`
- 远程仓库：`origin`
- 推送命令：`git push origin master`

### 3. 冲突处理
- 如果遇到冲突，使用 `git pull` 先拉取最新代码
- 解决冲突后再提交和推送
- 参考：`GIT_PULL_HANDLE_PROMPT.md`
